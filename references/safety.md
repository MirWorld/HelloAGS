# 安全与合规（G9 细则）

目标：把高风险行为前置识别、前置对齐、前置降风险；无法降风险时必须暂停并请求确认。

---

## 1) EHRB 信号识别（满足任一即视为高风险）

- **生产环境操作**：域名/数据库/环境包含 `prod/production/live`
- **PII/敏感数据**：姓名、证件号、手机号、邮箱、地址、生物特征、账户凭据
- **破坏性操作**：删除/清空/重建（如 `rm -rf`、`DROP TABLE`、`TRUNCATE`、无备份删库）
- **不可逆操作**：无回滚策略的数据库变更、无灰度的 API 发布、批量数据修正
- **权限变更**：角色提升、ACL/鉴权逻辑修改、越权风险
- **支付/金额**：订单金额、支付流程、对账逻辑
- **外部服务**：第三方 API、消息队列、缓存清空、跨系统同步
- **AI 工具链供应链**：新增/升级包、MCP 服务、Skill 脚本、hook、外部 CLI、OAuth 应用、模型代理库、持有高权限 token 的工具

---

## 2) 强制安全要求

- 禁止连接未授权的生产服务或拉取敏感数据
- 禁止明文保存密钥/令牌/账号到仓库或日志
- 依赖升级/新增：记录版本、兼容性验证、关注已知漏洞（CVE）
- 破坏性动作前必须备份，并优先提供 `--dry-run`/预演方案
- 对 LLM 推荐的包名/工具名默认不可信：必须核对官方源、项目既有配置或锁文件，防止幻觉包名 / slopsquatting 风险
- Skill / hook / MCP / 外部 CLI 脚本必须把输入 payload 当数据处理，禁止执行动态内容或拼接执行用户文本
- 涉及 OAuth/token/云账号权限时，必须记录最小权限、授权范围、回收/轮换方式；无法确认则暂停并请求确认

## 2.1) AI 工具链供应链检查（触发式）

触发条件（任一即执行）：
- 新增/升级依赖、安装外部 CLI、引入模型代理库
- 接入 MCP、Skill、hook、OAuth 应用、云端 Agent 或第三方自动化平台
- 工具会读取密钥、环境变量、仓库内容、浏览器/账号授权或生产配置

最小检查：
- 来源：是否来自官方源、既有仓库配置、锁文件或可信发布渠道
- 权限：需要哪些 token/OAuth scope/文件系统/网络权限，是否最小化
- 行为：是否会执行动态内容、远程脚本、用户输入或 assistant message
- 回滚：如何撤销授权、移除依赖、轮换 token 或禁用 hook/MCP
- 验证：至少有一条可复现的安装/运行/禁用检查

---

## 3) 风险处理优先级（默认）

1. **先降风险**：最小权限、最小影响面、灰度发布、可回滚
2. **再对齐**：把风险与代价写进“对齐摘要/风险评估”，明确成功标准与不可接受结果
3. **仍无法规避**：输出 `⚠️【HelloAGENTS】- 安全警告` 并暂停等待确认（格式见 `templates/output-format.md`）
